Amazon Web Services Migration

It's time to modernize.
Our current hosting website isn't secure and is costing us a lot of resources to implement, our websites will be tagged as insecure by modern web browsers such as Google Chrome. A secure website ensures that eServices are following the standard of the web regarding information security of our clients and stakeholders. Also this new hosting removes the language limit for MISA. Our current hosting only allows for PHP applications, which limits the pool of developers, and not all MISA members are equipped with this skill. AWS secures our websites, allows for multiple languages to be hosted, allowing our developers to shine, and is very cost effective.

AVP for Systems Documentation

For consistent eServices documents.
The Internal Systems Development position is going out and the Systems Documentation position is going in, mainly for two specific reasons. The Internals positions job can be easily be covered by templates by the cluster since the MISA almost always only requires registration systems and some information websites that usually come from the Events Cluster, the only thing that is different is the design. The new AVP position will be beneficial to the cluster and to MISA because we will now provide proper documentation before an external system will be deployed (useful for future redesigns or fixes) and MISA can keep track of it's clients properly.

eServices Version Control

cd eservs && git init
MISA and eServices will now harness the power of git through GitLab, having a central account where all repositories and projects will live (maybe forever). This will allow easy sharing of files between project teams and easy monitoring of progress by the VP for eServices and the AVP for Systems Documentation. Git also makes moving the files to the Linux cloud machine easier and therefore a much easy and more enjoyable deploying of our projects.
